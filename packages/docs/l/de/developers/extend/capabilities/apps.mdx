## Was sind Apps?

Apps ermöglichen es Ihnen, Aegis mit benutzerdefinierten Objekten, Feldern, Logikfunktionen, Frontend-Komponenten, KI-Fähigkeiten und mehr zu erweitern — alles als Code verwaltet. Anstatt alles über die UI zu konfigurieren, definieren Sie Ihr Datenmodell und Ihre Logik in TypeScript und stellen es in einem oder mehreren Workspaces bereit.

**Was Sie erstellen können:**

* **Benutzerdefinierte Objekte und Felder** — erweitern Sie Ihr Datenmodell mit neuen Entitäten oder fügen Sie bestehenden Objekten wie Company oder Person Felder hinzu
* **Logikfunktionen** — serverseitige Funktionen, die durch Datenbankereignisse, Cron-Zeitpläne oder HTTP-Routen ausgelöst werden
* **Frontend-Komponenten** — React-Komponenten, die innerhalb der UI von Aegis gerendert werden (Datensatzseiten, Befehlsmenü, Seitenpanels)
* **KI-Fähigkeiten und -Agenten** — erweitern Sie die KI von Aegis mit benutzerdefinierten Fähigkeiten
* **Ansichten und Navigation** — vorkonfigurierte gespeicherte Ansichten und Seitenleistenlinks

## Schnellstart

```bash filename="Terminal"
npx create-aegis-app@latest my-aegis-app
cd my-aegis-app
yarn aegis dev
```

Dies erstellt das Grundgerüst für eine neue App, startet optional einen lokalen Aegis-Server und beginnt, Ihre Dateien auf Änderungen zu überwachen. Den vollständigen Ablauf finden Sie im Leitfaden [Erste Schritte](/l/de/developers/extend/apps/getting-started).

## Wichtige Konzepte

### Entitätserkennung

Das SDK erkennt Entitäten, indem es Ihre TypeScript-Dateien nach Aufrufen von `export default define<Entity>({...})` scannt. Dateibenennung und Ordnerstruktur sind flexibel — die Erkennung ist AST-basiert, nicht pfadbasiert.

### Verfügbare Entitätstypen

| Funktion                           | Zweck                                            |
| ---------------------------------- | ------------------------------------------------ |
| `defineApplication()`              | Anwendungsmetadaten (erforderlich, eine pro App) |
| `defineObject()`                   | Benutzerdefinierte Objekte mit Feldern           |
| `defineField()`                    | Felder bei bestehenden Objekten                  |
| `defineLogicFunction()`            | Serverseitige Logik mit Triggern                 |
| `defineFrontComponent()`           | React-Komponenten in der UI von Aegis           |
| `defineRole()`                     | Berechtigungsrollen                              |
| `defineView()`                     | Konfigurationen gespeicherter Ansichten          |
| `defineNavigationMenuItem()`       | Navigationslinks in der Seitenleiste             |
| `defineSkill()`                    | Fähigkeiten von KI-Agenten                       |
| `defineAgent()`                    | KI-Agenten mit Prompts                           |
| `definePageLayout()`               | Benutzerdefinierte Layouts für Datensatzseiten   |
| `definePreInstallLogicFunction()`  | Wird vor der App-Installation ausgeführt         |
| `definePostInstallLogicFunction()` | Wird nach der App-Installation ausgeführt        |

### Entwicklungs-Workflow

1. **`yarn aegis dev`** — überwacht Quelldateien, baut bei Änderungen neu, synchronisiert mit dem Server, generiert typisierte API-Clients
2. **`yarn aegis build`** — erzeugt ein auslieferbares Build
3. **`yarn aegis deploy`** — stellt auf einem entfernten Aegis-Server bereit
4. **`yarn aegis add`** — erstellt interaktiv das Gerüst für eine neue Entität

### CLI-Referenz

```bash filename="Terminal"
yarn aegis help          # Alle Befehle auflisten
yarn aegis server start  # Lokalen Dev-Server starten
yarn aegis remote add    # Mit einem Aegis-Server verbinden
yarn aegis exec -n fn    # Eine Logikfunktion ausführen
yarn aegis logs -n fn    # Funktionsprotokolle streamen
```

Die vollständige CLI-Referenz finden Sie im Leitfaden [Erste Schritte](/l/de/developers/extend/apps/getting-started).
