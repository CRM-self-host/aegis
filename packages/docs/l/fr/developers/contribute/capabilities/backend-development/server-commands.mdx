## Commandes utiles

Ces commandes doivent être exécutées depuis le dossier packages/aegis-server.
Depuis n'importe quel autre dossier, vous pouvez exécuter `npx nx {command} aegis-server` (ou `npx nx run aegis-server:{command}`).

### Configuration initiale

```
npx nx database:reset aegis-server # setup the database with dev seeds
```

### Démarrer le serveur

```
npx nx run aegis-server:start
```

### Analyse

```
npx nx run aegis-server:lint # passez --fix pour corriger les erreurs de lint
```

### Test

```
npx nx run aegis-server:test:unit          # run unit tests
npx nx run aegis-server:test:integration   # run integration tests
```

Remarque: vous pouvez exécuter `npx nx run aegis-server:test:integration:with-db-reset` si vous avez besoin de réinitialiser la base de données avant de lancer les tests d'intégration.

### Réinitialiser la base de données

Si vous souhaitez réinitialiser et peupler la base de données, vous pouvez exécuter la commande suivante :

```bash
npx nx run aegis-server:database:reset
```

## Écosystème Tech

Aegis utilise principalement NestJS pour le backend.

Prisma a été le premier ORM que nous avons utilisé. Mais pour permettre aux utilisateurs de créer des champs et objets personnalisés, un niveau inférieur avait plus de sens car nous avons besoin d'un contrôle granulaire. Le projet utilise maintenant TypeORM.

Voici à quoi ressemble maintenant la pile technologique.

**Noyau**

* [NestJS](https://nestjs.com/)
* [TypeORM](https://typeorm.io/)
* [GraphQL Yoga](https://the-guild.dev/graphql/yoga-server)

**Base de données**

* [Postgres](https://www.postgresql.org/)

**Intégrations tierces**

* [Sentry](https://sentry.io/welcome/) pour suivre les bugs.

**Tests**

* [Jest](https://jestjs.io/)

**Outils**

* [Yarn](https://yarnpkg.com/)
* [Oxlint](https://oxc.rs/docs/guide/usage/linter.html)

**Développement**

* [AWS EKS](https://aws.amazon.com/eks/)
