## Что такое приложения?

Приложения позволяют расширять Aegis с помощью пользовательских объектов, полей, логических функций, фронтенд-компонентов, навыков ИИ и многого другого — всё это управляется как код. Вместо настройки всего через интерфейс вы определяете модель данных и логику на TypeScript и развёртываете её в одном или нескольких рабочих пространствах.

**Что вы можете создать:**

* **Пользовательские объекты и поля** — расширяйте модель данных новыми сущностями или добавляйте поля к существующим объектам, таким как Company или Person
* **Логические функции** — серверные функции, запускаемые событиями базы данных, расписаниями cron или HTTP-маршрутами
* **Фронтенд-компоненты** — компоненты React, которые отображаются в интерфейсе Aegis (страницы записей, командное меню, боковые панели)
* **Навыки и агенты ИИ** — расширяйте ИИ Aegis пользовательскими возможностями
* **Представления и навигация** — преднастроенные сохранённые представления и ссылки боковой панели

## Быстрый старт

```bash filename="Terminal"
npx create-aegis-app@latest my-aegis-app
cd my-aegis-app
yarn aegis dev
```

Это создаёт каркас нового приложения, при необходимости запускает локальный сервер Aegis и начинает отслеживать изменения в ваших файлах. См. руководство [Начало работы](/l/ru/developers/extend/apps/getting-started) для полного пошагового разбора.

## Ключевые понятия

### Обнаружение сущностей

SDK обнаруживает сущности, сканируя ваши файлы TypeScript в поисках вызовов `export default define<Entity>({...})`. Имена файлов и структура папок гибкие — обнаружение основано на AST, а не на путях.

### Доступные типы сущностей

| Функция                            | Назначение                                                   |
| ---------------------------------- | ------------------------------------------------------------ |
| `defineApplication()`              | Метаданные приложения (обязательно, по одному на приложение) |
| `defineObject()`                   | Пользовательские объекты с полями                            |
| `defineField()`                    | Поля в существующих объектах                                 |
| `defineLogicFunction()`            | Серверная логика с триггерами                                |
| `defineFrontComponent()`           | Компоненты React в интерфейсе Aegis                         |
| `defineRole()`                     | Роли доступа                                                 |
| `defineView()`                     | Конфигурации сохранённых представлений                       |
| `defineNavigationMenuItem()`       | Ссылки боковой панели навигации                              |
| `defineSkill()`                    | Навыки агента ИИ                                             |
| `defineAgent()`                    | ИИ-агенты с промптами                                        |
| `definePageLayout()`               | Пользовательские макеты страниц записей                      |
| `definePreInstallLogicFunction()`  | Выполняется перед установкой приложения                      |
| `definePostInstallLogicFunction()` | Выполняется после установки приложения                       |

### Рабочий процесс разработки

1. **`yarn aegis dev`** — следит за исходными файлами, пересобирает при изменениях, синхронизируется с сервером, генерирует типизированные клиенты API
2. **`yarn aegis build`** — создаёт дистрибутивную сборку
3. **`yarn aegis deploy`** — развёртывает на удалённый сервер Aegis
4. **`yarn aegis add`** — интерактивно создаёт каркас новой сущности

### Справочник по CLI

```bash filename="Terminal"
yarn aegis help          # Показать все команды
yarn aegis server start  # Запустить локальный сервер разработки
yarn aegis remote add    # Подключиться к серверу Aegis
yarn aegis exec -n fn    # Выполнить логическую функцию
yarn aegis logs -n fn    # Транслировать логи функции
```

См. руководство [Начало работы](/l/ru/developers/extend/apps/getting-started) для полной справки по CLI.
